## 🎯 Цель проекта
• **Какую задачу решает приложение:** Демонстрация работы различных типов сервисов (Service) в ОС Android.  
• **Почему эта задача важна:** Понимание жизненного цикла и способов взаимодействия с фоновыми задачами (Started Service, Bound Service, JobService) является критически важным для разработки эффективных Android-приложений.  
• **Целевая аудитория:** Android-разработчики, студенты, изучающие мобильную разработку.  
________________________________________

## 📌 Требования к приложению
### Функциональные требования
• Запуск и остановка музыкального сервиса в фоновом режиме.  
• Реализация таймера, работающего независимо от жизненного цикла Activity.  
• Получение данных от сервиса через привязку (Binding).  
• Подсчет шагов с использованием датчиков устройства.  
• Очистка кэша при запуске/остановке приложения.  

### Нефункциональные требования
• **Производительность:** Минимальное потребление ресурсов в фоновом режиме.  
• **Удобство использования:** Простой интерфейс с кнопками управления для каждого сервиса.  
• **Надёжность:** Корректная обработка остановки сервисов и освобождение ресурсов (например, MediaPlayer).  
________________________________________

## 🧠 Общая архитектура
• **Выбранный подход:** Стандартная компонентная архитектура Android (Activity + Services).  
• **Почему выбран именно он:** Это наиболее прямой способ продемонстрировать механизмы работы системных компонентов Android без наслоения дополнительных паттернов (таких как MVVM), что упрощает обучение.  
• **Основные компоненты системы:**
  - `MainActivity`: Интерфейс управления.
  - `MusicService`: Фоновое воспроизведение.
  - `TimerService`: Привязанный сервис для отсчета времени.
  - `StepCounterService`: Сервис для работы с сенсорами.
  - `CacheCleanerService`: Сервис для выполнения кратковременных задач.

## 🧩 Подробное описание классов
### 📌 Класс: MainActivity
- **Роль:** Главный экран приложения.
- **Ответственность:** Управление жизненным циклом сервисов через пользовательский ввод (кнопки, переключатели). Отображение состояния сервисов.
- **Основные методы:**
  - `onCreate()` — инициализация UI и слушателей.
  - `updateTimer()` — обновление текста таймера в UI из фонового потока.
  - `initComponents()` — настройка ServiceConnection.
- **Взаимодействие:** Запускает/останавливает сервисы через `Intent`, привязывается к `TimerService`.

### 📌 Класс: MusicService
- **Роль:** Фоновое воспроизведение музыки.
- **Методы:** `onStartCommand` запускает плеер, `onDestroy` освобождает ресурсы.

### 📌 Класс: TimerService
- **Роль:** Bound Service (привязанный сервис).
- **Почему используется:** Позволяет Activity получать актуальное количество секунд напрямую через Binder.
________________________________________

## 🔄 Диаграмма работы приложения
1. Пользователь нажимает "Start TS".
2. `MainActivity` запускает и привязывается к `TimerService`.
3. `TimerService` начинает отсчет.
4. `MainActivity` через `updateTimer` запрашивает данные у сервиса каждую секунду.

## ⚙️ Работа с потоками
• **Используются:** `Thread` (в MainActivity для обновления UI).  
• **Почему:** Для демонстрации необходимости обновления UI из неосновного потока при работе с долгоживущими задачами.  
• **Предотвращение зависаний:** Работа с датчиками и медиа-плеером вынесена в сервисы.  
________________________________________

## 💾 Работа с данными
• **Где хранятся:** `SharedPreferences`.  
• **Почему:** Для сохранения простых состояний (количество шагов, последнее время таймера) между запусками.
